수능 Sentinel - Curated Workbench 

Front end 

Three-pane dashboard 
Source / Extraction / Item Transformation 

여기에서 Extraction 에 대한 feedback를 ai에게 주어 학습시킬 수 있어야 함 

Pane 1 : The Search & Filter 
Inputs: Keyword, Item type (Blank, Order , etc) 
Visual : A "Crawler Progress" bar showing the search across repositories (JSTOR, arXiv, etc.)

여기에서, 어떤 기준으로 지문을 찾아야 하는지 내가 "알맞는 지문"에 대한 기준과 예시로 데이터를 제공하여 미리 모델을 학습시킬것임  

무조건 verbatim으로 골라내어야 하면 모델의 환각 등으로 인해 원문 소스에 존재하지 않는 워딩/문장 등을 제공하지 않도록 강력히 박아두어야함 


Pane 2: 
Pane 1에서 결정되어진 지문들을 보여주는 공간으로, 각 지문의 소스 url을 명시해야하며, 골라낸 지문의 부분이 몇 글자인지도 적어주고,  

언제든지, Pane 1에서 crawl되고 선택되어 pane 2에서 표출된 지문이 수능 문항 지문으로 쓰이기에 적합한지 여부에 대한 피드백을 내가 모델에게 주어 모델이 이를 학습해주어야 함 
궁극적으로는, 지속적인 피드백과 학습을 통하여, 수능영어 지문으로 가장 적합하고, 각 유형별로 적합한지를 정확히 모델이 스스로 집어낼 줄 알게 되어야함 

그를 위하여 Pane 2에서 표출된 지문 각각에 대하여 피드백을 선택적으로 작성하고 학습데이터로 보낼 수 있을 창이 필요함   

이 과정에서, 지문 별로 적합도/퀄리티 지수를 점수로 표기 하는게 좋을지 고민중. 한다면 어떤 metric으로 어떻게 점수화 시킬지도 아직 고민중. 이때, 어떤 유형에 걸맞는 지문인지에 대해서도 표기가 필요함. 적합한 유형은 한개 이상일 수도 있음. 
모델이 점수를 알아서 매겨주면.. 내가 그 점수를 수정하여 재입력 시켜줄 수 있어야 할거같음. 그래야 향후에 높은 점수에 걸맞는 지문을 찾아올테니. 이보다 좋은 방식이 있다면 open to discussion.  



 

Pane 3: 
적합한 지문으로 필터링 된 것으로, 실제 출제 문항의 형태로 완성시켜주는 공간. 
일차적으로 당연히 Pane 2에서 필터링이 된 것들에 대하여만 문항을 생성하여야 함. 
이 과정에서, 사용자가 pane 2에 표출된 것들 중 선택하여 일부에 대하여만 생성할 수도 있어야 하고, select all, 자동으로 전부 문제화 시키기 등 기능에 대한 선택도 가능해야함 

모델이 만들어낸 문항에 대한 평가도 사용자가 feedback 을 입력하여 모델이 학습하여 향후 좋은 문항을 만들어 낼 수 있도록 해야함. 


Pane 4? 
한 페이지안에 전부 표시가 가능한진 모르겠지만 
정해진 양식에 따라서, 완성된 문항에 대한 해설을 작성해주는 기능이 필요함. 
그 정해진 양식에 대한 데이터는 향후 제공하여 모델이 학습하도록 할것임. 
이것은 후순위 작업이지만 기능 추가될것은 미리 고려해두어야함
